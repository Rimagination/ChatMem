# ChatMem 中文化与界面收敛设计

**Date:** 2026-04-08

## Goal

在不改变 ChatMem 现有双栏结构和核心交互的前提下，对界面做一次保守收敛的视觉优化：

- 保留当前白底、浅灰背景、轻科技感的总体气质
- 将全局主色从蓝色替换为低饱和、低亮度的绿色体系
- 系统性缩小过大的圆角，减少“鼓起来”的观感
- 重做顶部复制操作区和刷新按钮的样式，让布局更规整
- 将 ChatMem 品牌下方英文副文案改为中文，保持中文软件一致性

## Context

用户当前明确指出了四类问题：

1. 不同版块的圆角矩形角度过大，整体偏软、偏鼓
2. `复制位置`、`复制恢复命令` 这类按钮排版不协调，窄一些时换行难看
3. `ChatMem` 下方副文案是英文，不符合中文软件定位
4. `刷新` 按钮当前只是普通次级按钮，样式不够精致

用户同时给出了两个明确约束：

- 继续保留现有蓝白科技感的底子，不做重设计
- 用用户提供的绿色作为方向参考，但实际落地要更克制，饱和度和亮度都不能太高

## Explored Approaches

### Approach 1: 轻量换色

只替换主色和文案，尽量不碰结构与排版。

优点：

- 风险最低
- 改动最小

缺点：

- 圆角过大和复制按钮不协调的问题只能局部缓解
- 最终观感提升有限

### Approach 2: 保守收敛

保留现有布局和玻璃感，仅做配色、圆角、关键操作区和文案的体系化收敛。

优点：

- 与当前产品最连续
- 能同时解决配色、圆角、按钮和中文化问题
- 风险可控，适合一次完成

缺点：

- 不是“焕然一新”的重设计

### Approach 3: 桌面工具化

进一步减少渐变、阴影和圆角，让产品更偏专业工具。

优点：

- 工具感更强
- 结构更硬朗

缺点：

- 风格变化过大
- 会偏离用户“保留现有底子”的要求

## Selected Product Decision

本次采用 **Approach 2: 保守收敛**。

产品目标不是换风格，而是把现有界面收紧、修正、中文化，让它从“轻漂亮”变成“更稳、更顺眼、更像桌面工具”。

## Visual Direction

### Visual Thesis

保留明亮桌面工作区、浅玻璃感和低噪音背景层次，但把视觉语言从“蓝色圆润卡片”收敛到“白底工具界面 + 克制绿色强调”。

### Color Thesis

绿色作为全局主色，但只能在真正需要强调的地方出现：

- 选中态
- 焦点态
- 主操作
- 图标型次操作的 hover / active 态
- 局部高亮信息

背景、分割、正文文本仍然以中性灰白为主，不做大面积绿色铺陈。

### Radius Thesis

圆角收紧必须是体系性的，而不是只改一处：

- 侧栏与主工作区外层容器
- 顶部信息条
- 统计块
- 消息气泡
- 文件变更块
- 弹窗与单选项容器
- 按钮与输入框

所有这些区域都需要同步缩小一档，避免视觉语言不一致。

## Color System Specification

本次实现中，绿色基准直接写入前端变量，使用如下基准值：

- `--accent: #2a9d62`
- `--accent-strong: #1f7d4e`
- `--accent-soft: rgba(42, 157, 98, 0.14)`

约束：

- 不允许把用户截图中的高亮绿直接照搬为全局主色
- hover 和 focus 态只允许略提亮，不允许变成荧光绿
- 绿色阴影透明度低于当前蓝色版本，避免按钮过于跳出

`success` 系列保留独立语义，但色相需要与新的绿色主色保持同一方向，避免出现“一个界面里两种不相关的绿”。

## Radius System Specification

本次统一收敛为以下圆角体系：

- `--radius-sm: 8px`
- `--radius-md: 12px`
- `--radius-lg: 16px`
- `--radius-xl: 22px`

映射规则：

- 侧栏与主工作区外框使用 `22px`
- 信息条、文件变更区、大块浅底容器使用 `16px`
- 输入框、常规按钮、列表项、tool call 容器使用 `12px`
- 小标签、状态 pill 保持胶囊形，不受本次缩角约束

消息气泡额外规则：

- 助手气泡圆角固定为 `16px`
- 用户气泡保留轻微不对称，但右下角仍要更利落，避免太圆

## Key UI Changes

### 1. 品牌副文案中文化

`ChatMem` 下方副文案改为：

`本地对话记录，一处查看，随时续接`

要求：

- 不使用英文
- 不写营销话术
- 语气偏工具型而非宣传型

### 2. 顶部辅助英文清理

除产品名和 agent 名本身外，顶部可见文案统一转为中文。

例如当前类似 `Conversation` 的辅助说明需要改成中文表述，如：

- `CLAUDE 对话`
- `CODEX 对话`
- `GEMINI 对话`

要求整个首屏不再出现无必要英文说明。

### 3. 刷新按钮重做

刷新按钮改为轻量工具按钮，而不是普通次级按钮。

设计要求：

- 结构为“图标 + 文字”
- 视觉权重低于主标题
- 尺寸略小于常规操作按钮
- 默认状态为浅底或透明底
- hover 时使用极轻的绿色提示，不做厚重描边

布局要求：

- 继续放在左侧栏顶部右上角
- 在桌面端不抢占标题视觉中心
- 在窄宽度下优先保证按钮本身完整，不拆字

### 4. 顶部复制操作区重排

当前 `复制位置` 和 `复制恢复命令` 的问题不是逻辑，而是按钮过于普通、标签过长、排列不稳。

本次改成“信息在左，动作在右”的紧凑操作条：

- 左侧为标签 + 路径值
- 右侧为动作组
- 动作组内部两个按钮使用统一高度和最小宽度

动作组规则：

- 两个按钮必须保持单行文字，不允许按钮内部断行
- 使用 `white-space: nowrap`
- 每个按钮设置稳定的 `min-width`
- 桌面宽度优先横向排列
- 中等宽度允许整个动作组换到下一行，但动作组内部仍保持整齐

视觉规则：

- 两个复制按钮采用更轻、更贴近工具条的样式
- 保留现有复制成功 / 失败的即时反馈逻辑
- 不新增弹窗，不使用浏览器 alert

### 5. 信息条层级优化

顶部元信息条保留现有功能，但层级要更清楚：

- 标签层更弱
- 路径值是主信息
- 操作组是清晰但不过度突出的辅助动作

路径值要求：

- 允许长路径换行
- 不让路径文本与按钮争抢同一行
- 不可用状态继续保留 muted 样式

### 6. 整体块面收敛

下列区域同步减少“卡片感”：

- 左侧栏容器
- 右侧工作区容器
- 统计块
- 文件变更区
- tool call 容器
- 弹窗单选项卡片

目标不是删掉层次，而是减少“圆、鼓、浮”的感觉。

## Frontend Scope

### Expected Files

- `src/App.tsx`
- `src/styles.css`
- `src/__tests__/App.test.tsx`

根据实现情况，允许小范围更新：

- `src/__tests__/ConversationDetail.test.tsx`

本次尽量不改动：

- 后端数据结构
- 迁移逻辑
- 删除逻辑
- 会话列表筛选逻辑

## Behavior Constraints

- 复制位置与复制恢复命令的实际逻辑保持不变
- 成功 / 失败按钮反馈机制保持不变
- 不新增新的全局状态
- 不新增主题切换功能
- 不改变现有双栏布局和信息架构

## Testing Strategy

实现完成后至少验证：

1. 主色变量已从蓝色切换到低饱和绿色体系
2. 品牌副文案为中文
3. 顶部刷新按钮仍可正常触发原有刷新逻辑
4. `复制位置` 与 `复制恢复命令` 按钮在常见桌面宽度下不出现按钮内换行
5. 现有与复制操作相关的测试在更新文案后继续通过
6. `npm run build` 能成功完成

## Acceptance Criteria

- 界面整体仍然明显属于原有 ChatMem，而不是一套新设计
- 主色从蓝色切换为克制绿色，且不刺眼
- 大圆角问题明显缓解，版块观感更利落
- `ChatMem` 下方副文案变为中文
- 刷新按钮视觉上明显比原先更精致、更像工具栏动作
- 顶部复制操作区在桌面宽度下更整齐，不再出现难看的按钮断行感
- 本次改动不影响现有复制、刷新、选择会话等行为
